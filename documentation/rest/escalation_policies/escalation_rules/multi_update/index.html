PUT escalation_policies/:escalation_policy_id/escalation_rules

Updates the entire collection of escalation rules for an existing escalation policy. The ordering is determined by the positions of each rule in the array. To create a new rule, add it to the array without an id. To remove a rule, do not include it in the array.

Resource URL 

PUT https://<subdomain>.pagerduty.com/api/v1/escalation_policies/:escalation_policy_id/escalation_rules

Parameters

Name Type Required Description
escalation_rules Array Yes An ordered array of escalation rules. The parameters can be found under the Escalation Rules API.

Example

Update all escalation rules 

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X PUT \
    -d '{
      "escalation_rules": [
        {
          "escalation_delay_in_minutes": 12,
          "targets": [
            {
              "type": "schedule",
              "id": "PWEVPB6"
            }
          ]
        },
        {
          "id": "PNTPYK0",
          "escalation_delay_in_minutes": 24
        }
      ]
    }' \
    "https://acme.pagerduty.com/api/v1/escalation_policies/PBZ3BZV/escalation_rules"

Note: Content-type of the request must be application/json
