POST escalation_policies/:escalation_policy_id/escalation_rules

Creates a new escalation rule for an escalation policy and appends it to the end of the existing escalation rules.

Resource URL 

POST https://<subdomain>.pagerduty.com/api/v1/escalation_policies/:escalation_policy_id/escalation_rules

Parameters

Name Type Required Description
escalation_delay_in_minutes Integer Yes The escalation timeout in minutes. If an incident is not acknowledged within this timeout then it will escalate onto the next escalation rule.
rule_object Object Yes An object containing the schedule or user id and type.

Rule Object Parameters

Name Type Required Description
type String Yes A string of either schedule or user,
id String Yes The id of the schedule or user assigned to this rule.

Escalation Rule Parameters

Name Type Required Description
escalation_delay_in_minutes Integer Yes The escalation timeout in minutes. Must be at least 5 if the rule has multiple targets, and at least 1 if not. If an incident is not acknowledged within this timeout then it will escalate onto the next escalation rule.
rule_object Object No
This field is deprecated. See targets instead
A single target. Ignored if targets is provided.
targets Array Yes The array of the targets an incident should be assigned to upon reaching this rule. Parameters detailed below.

Target Parameters

Name Type Required Description
type String Yes A string of either schedule or user,
id String Yes The id of the schedule or user assigned to this rule.

Example

Create an escalation rule 

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X POST \
    -d '{"escalation_rule":{"escalation_delay_in_minutes":10,"targets":[{"type":"user","id":"PPSFHH7"}]}}' \
    "https://acme.pagerduty.com/api/v1/escalation_policies/PBZ3BZV/escalation_rules"

Note: Content-type of the request must be application/json
