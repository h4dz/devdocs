POST services/:service_id/email_filters

Create a new Email Filter for the specified service.

Resource URL 

POST https://<subdomain>.pagerduty.com/api/v1/services/:service_id/email_filters

Parameters

Name Type Required Description
subject_mode String No One of always, match, no-match, which, respectively, means to not filter the email trigger by subject, filter it if the email subject matches the given regex, or filter if it doesn't match the given regex. Defaults to always.
subject_regex String No The regex to be used when subject_mode is match or no-match. It is a required parameter on such cases.
body_mode String No One of always, match, no-match, which, respectively, means to not filter the email trigger by body, filter it if the body email matches the given regex, or filter if it doesn't match the given regex. Defaults to always.
body_regex String No The regex to be used when body_mode is match or no-match. It is a required parameter on such cases.
from_email_mode String No One of always, match, no-match, which, respectively, means to not filter the email trigger by its from address, filter it if the email from address matches the given regex, or filter if it doesn't match the given regex. Defaults to always.
from_email_regex String No The regex to be used when from_email_mode is match or no-match. It is a required parameter on such cases.

Example 

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X POST \
    -d '{"email_filter":{"body_mode":"no-match","body_regex":"sev 3"}}' \
    "https://acme.pagerduty.com/api/v1/services/P9312OW/email_filters"

Note: Content-type of the request must be application/json

Response Status: 201 Created 

{
  "email_filter": {
    "subject_mode": "always",
    "subject_regex": null,
    "body_mode": "no-match",
    "body_regex": "sev 3",
    "from_email_mode": "always",
    "from_email_regex": null,
    "id": "PM1H1PY"
  }
}
