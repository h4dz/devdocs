GET reports/incidents_per_time

Get high level statistics about the number of incidents created for the desired time period, summed daily, weekly or monthly.

Resource URL

GET https://<subdomain>.pagerduty.com/api/v1/reports/incidents_per_time/

Parameters

NameTypeRequiredDescription
sinceDateYesThe start of the date range over which you want to search. The time element is optional.
untilDateYesThe end of the date range over which you want to search. This should be in the same format as since.
rollupStringNo

Possible values are daily, weekly or monthly. Specifies the bucket duration for each summation. Defaults to monthly.

Example: A time window of two years (based on since and until) with a rollup of monthly will result in 24 sets of data points being returned (one for each month in the span).

Example

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X GET -G \
    --data-urlencode "since=2012-08-01T00:00:00Z" \
    --data-urlencode "until=2012-08-05T00:00:00Z" \
    --data-urlencode "rollup=daily" \
    "https://acme.pagerduty.com/api/v1/reports/incidents_per_time/"

Note: Content-type of the request must be application/json
