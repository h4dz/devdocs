GET users/:user_id/log_entries

List all incident log entries that describe interactions with a specific user.

Resource URL

GET https://<subdomain>.pagerduty.com/api/v1/users/:user_id/log_entries

Parameters

This action is paginated. See the pagination documentation for details.
NameTypeRequiredDescription
time_zoneTime ZoneNoTime zone in which dates in the result will be rendered. Defaults to UTC
sinceDateNoThe start of the date range over which you want to search.
untilDateNoThe end of the date range over which you want to search.
is_overviewBooleanNoIf true, will only return log entries of type trigger, acknowledge, or resolve. Defaults to false.
includeArrayNoArray of additional details to include. This API accepts channel, incident, and service. If channel is not included, a summary will be returned (See channel format information for more information.)

Response Fields

NameTypeDescription
log_entriesArrayArray of the log entries. See the Log Entry Show API for details.
totalIntegerTotal number of log entries matching the request ignoring pagination.
offsetIntegerEchoes offset pagination parameter.
limitIntegerEchoes limit pagination parameter.

Example

Example

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X GET -G \
    "https://acme.pagerduty.com/api/v1/users/PPSFHH7/log_entries"

Note: Content-type of the request must be application/json
