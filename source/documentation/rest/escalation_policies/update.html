PUT escalation_policies/:id

Updates an existing escalation policy and rules.

Resource URL

PUT https://<subdomain>.pagerduty.com/api/v1/escalation_policies/:id

Parameters

NameTypeRequiredDescription
nameStringNoThe name of the escalation policy.
repeat_enabledBooleanNoWhether or not to allow this policy to repeat its escalation rules after the last rule is finished.
num_loopsIntegerNoThe number of times to loop over the set of rules in this escalation policy.
escalation_rulesArrayNoThe escalation rules for this policy. The ordering and available parameters are found under the Escalation Rules API.

Example

Update an escalation policy

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X PUT \
    -d '{"name":"Escalation Policy 2"}' \
    "https://acme.pagerduty.com/api/v1/escalation_policies/PBZ3BZV"

Note: Content-type of the request must be application/json
