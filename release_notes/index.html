October 3, 2014
  • Support adding and changing job_title of users
June 16, 2014
  • Support for HTTP Basic Authentication will no longer be provided. Do not use HTTP Basic Authentication for any new PagerDuty API integrations.
March 4, 2014
  • New on-call endpoints are available for users and escalation policies.
August 19, 2013
  • New incident endpoints for changing state are available.
August 1, 2013
  • Webhooks are now generally available.
July 11, 2013
  • Webhooks (Limited Release) can now be queried through the API.
June 7, 2013
  • Escalation Policies can now be queried through the API.
June 1, 2013
  • Responses from the Users API can now optionally return notification rules and contact methods for users.
April 8, 2013
  • Incident Notes can now be queried and created through the API.
February 01, 2013
  • The timezone which response datetimes are rendered in for Log Entry queries can now be specified.
January 30, 2013
  • Incidents#show Api released.
January 28, 2013
  • Contact Method and Notification Rules APIs now show push notifications properly.
December 7, 2012
  • Alerts API now has filter parameter properly documented.
December 6, 2012
  • User#create has requester_id properly documented.
November 26, 2012
  • Email Filters, used by email services, can now be queried and manipulated through the API.
November 20, 2012
  • Log Entries can now be queried through the API.
November 19, 2012
  • Services can now be queried and manipulated through the API.
November 16, 2012
  • The since and until parameters for Incidents#list request are now properly documented as being not required
October 29, 2012
  • Showing requester_id parameter on curl examples, when they are required (which only happens when using the Api Key authentication mechanism, rather than basic authentication). Affected docs : Incidents#update and MaintenanceWindows#create.
October 22, 2012
  • Showing requester_id as a top level parameter on Incidents#update and MaintenanceWindows#create.
September 28, 2012
  • Fixing wrong date format in schedules examples on Schedules#show and Schedules#users.
September 21, 2012
  • New APIs released for Maintenance Windows, allowing you to disable one or more services for a predetermined period of time.
September 18, 2012
  • Documented how API Keys can be used instead of HTTP Authentication.
September 18, 2012
  • Schedules rotation_virtual_start property on Schedules#create, Schedules#preview and Schedules#update is now correctly documented as being the Date type.
September 12, 2012
  • Fixed GET request curl formatting
  • Better examples on Incidents#list
September 5, 2012
  • Improved error handling and messaging for multi-incident requests on Incidents#update.
August 27, 2012
  • developer.pagerduty.com site is live.
  • New V1 APIs released for Alerts, Reports, Schedules and Users.
